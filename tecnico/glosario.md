# Glosario y Recursos

Para que todos hablemos el mismo idioma técnico y entendamos la filosofía detrás de los **Open Badges**, es vital dominar estos conceptos.

## Glosario de Términos

| Término                    | Definición                                                                                                                                             |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Assertion (Afirmación)** | El corazón del estándar. Es el registro individual que vincula a un receptor con una insignia específica. Contiene la evidencia y la fecha de emisión. |
| **BadgeClass**             | El "molde" o plantilla de la insignia. Define el nombre, la descripción, los criterios para obtenerla y la imagen base.                                |
| **Issuer (Emisor)**        | La entidad (como el GDG Guadalajara) que tiene la autoridad para otorgar el reconocimiento.                                                            |
| **Baking (Horneado)**      | El proceso de incrustar los metadatos de la *Assertion* dentro de los bits de una imagen (PNG o SVG).                                                  |
| **Baked Image**            | Una imagen que "carga" su propia prueba de autenticidad. No es solo un dibujo; es un certificado digital autónomo.                                     |
| **JSON-LD**                | *JSON for Linked Data*. El formato de datos que usamos para que las máquinas entiendan las relaciones entre emisores y receptores.                     |
| **Backpack (Mochila)**     | Plataformas externas (como Badgr o Canvas) donde los usuarios pueden coleccionar, organizar y compartir sus insignias horneadas.                       |
| **Verification**           | El proceso mediante el cual un tercero valida que los metadatos de la imagen son legítimos consultando la URL del emisor.                              |
